    Coding Challenge
    Projects

    Coding Challenges are designed to test students' coding abilities and problem-solving skills. These challenges are intended to simulate real-world coding scenarios and provide students with the opportunity to practice their coding skills in a supportive environment. Students are also given feedback on their performance, which helps them to improve their coding skills over time.

    1

    Leveled Challenges

    Tackle challenges based on difficulty categories - Easy, medium, and hard that will challenge your skills in both algorithmic and foundational knowledge of software engineering.
    2

    Post-Challenge Discussion

    Participate in a discussion about the logic, steps, and common mistakes associated with the challenge. This discussion helps students to understand the problem-solving process, gain valuable insights into how experts approach coding challenges, and learn from their peers and gain a deeper understanding of the problem-solving process

    During NEXT, students will do 1 project until completion and the role of the students in NEXT is to develop from the provided PRD. In the process of development, the project is divided into several milestones corresponding to one sprint.

    Milestone 1: Github Profile Creation (1 Week)

    Define the objectives of enhancing your GitHub profile and understanding the value it brings to your professional image.
    Research and analyse impressive GitHub profiles of industry professionals and top developers to gather inspiration and ideas.
    Develop a clear plan for optimising your GitHub profile, including updating your profile picture, bio, and contact information.
    Review and refine your existing repositories, ensuring they are well-organised, have clear READMEs, and contain meaningful commit messages.
    Create a new README for your GitHub profile to showcase your skills, projects, and contributions.
    Set up a version control system for your GitHub profile repositories and consider using a project management tool (e.g., GitHub Projects) for tracking improvements.
    Customise your GitHub profile with a personalised README and other visual elements like GitHub stats, activity graphs, and pinned repositories.
    Integrate social media links, personal websites, and online portfolios, if applicable, to provide a comprehensive view of your professional identity.

    Milestone 2: Minimum Viable Product (2 Weeks)

    Define the core features and functionalities required for the MVP.
    Develop the essential components and user interfaces for the MVP.
    Implement basic user authentication and data storage mechanisms.
    Perform initial testing to ensure the MVP meets the specified requirements.

    Milestone 3: Feature Development (1 Week)

    Plan and prioritise additional features to be added to the project.
    Begin implementing new features based on the project roadmap.
    Test and validate each feature as it's completed to ensure functionality.

    Milestone 4: Feature Development II (2 Weeks)

    Continue implementing new features and functionalities.
    Integrate any necessary APIs or external services including openAI API into the project.
    Conduct regular code reviews to maintain code quality and consistency.

    Milestone 5: Performance Optimization/Performance Monitoring (1 Week)

    Conduct performance analysis to identify bottlenecks in the application.
    Optimise database queries and indexes for better efficiency.
    Implement caching mechanisms to improve response times.
    Profile and optimise critical sections of code for better overall performance.
